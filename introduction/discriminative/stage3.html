為防止對同一頭豬「重複計數」，推論時必須掛載目標追蹤演算法（Tracking），為每頭豬賦予唯一 ID。陣列內部含有多少個獨立 ID，即代表當前畫面有幾頭豬。點擊左側步驟查看對應實作。

✅ 完整程式
import cv2
from ultralytics import YOLO

model = YOLO('best.pt')
cap = cv2.VideoCapture('drone_stream.mp4')

while cap.isOpened():
    success, frame = cap.read()
    if not success: break

    results = model.track(frame, persist=True,
                          tracker="bytetrack.yaml")

    if results[0].boxes.id is not None:
        ids = results[0].boxes.id.int().cpu().tolist()
        cv2.putText(frame, f"Pigs: {len(ids)}",
                    (30, 50), cv2.FONT_HERSHEY_SIMPLEX,
                    1, (0, 255, 0), 2)

    cv2.imshow("Census", results[0].plot())
    if cv2.waitKey(1) & 0xFF == ord('q'): break

cap.release()
cv2.destroyAllWindows()
📋 詳細資訊

讀取影像來源

OpenCV 的 VideoCapture 是統一接口，支援多種來源：

  • 本地檔案：傳入 mp4 / mov 等路徑，方便離線測試演算法正確性。
  • RTSP 串流：傳入 rtsp://… URL，可即時接收無人機 / IP cam 影像。
  • USB Webcam：傳入整數 (0, 1, …) 代表本機攝影機編號。
  • HTTP MJPEG：許多消費級無人機提供 http://drone.ip/video 介面。

逐幀 YOLO 偵測

呼叫 model(frame) 會回傳一個 Results 物件，重要屬性：

  • boxes.xyxy：所有 bbox 的 (x1, y1, x2, y2) 像素座標。
  • boxes.conf：信心分數，可用 conf=0.5 過濾低信心目標。
  • boxes.cls：類別索引（本專案皆為 0）。
  • 速度：桌機 RTX 3060 上 YOLOv11n @ 640px 約 5~8ms/frame。

ByteTrack 跨幀關聯

單純逐幀偵測會讓「同一頭豬」在每幀變成不同實例，導致重複計數。ByteTrack 透過動態關聯解決此問題：

  • 運動預測：使用 Kalman Filter 預測下一幀位置。
  • IoU 匹配：將預測位置與當前偵測框比對，相近者視為同一目標。
  • 低信心保留：不直接丟棄低信心框，給予二次匹配機會以對抗短暫遮擋。
  • persist=True：關鍵參數，告訴 YOLO 在連續呼叫間保留 tracker 狀態。

取得唯一 ID 清單

每個被追蹤的目標都會獲得整數 ID，新進場的目標 ID 持續遞增：

  • boxes.id：形狀為 (N,) 的 tensor，可能為 None（無偵測時）。
  • .int().cpu().tolist()：轉為 Python list 便於後續處理。
  • 典型輸出：[3, 7, 12, 15, 18]，代表畫面有 5 頭豬，ID 不連續是因為其他豬已離開畫面。
  • None 防呆：務必先檢查 boxes.id is not None，否則會 AttributeError。

計算豬隻數量

不同的「數量」定義對應不同實作：

  • 當前畫面數：len(track_ids)，最直觀。
  • 累計過去看過數：維護一個 set()，每幀 set.update(ids)，最後 len(set) 即為總數。
  • 跨越虛擬線數：適合「進/出計數」，需追蹤每個 ID 的中心點軌跡是否跨越預設邊界。
  • 實務挑戰：豬隻長時間遮擋可能讓 tracker 分配新 ID，導致重複計入；可調整 track_buffer 緩解。

繪製結果疊圖

常用 OpenCV API：

  • cv2.putText：寫入文字，最後參數是線條粗細。
  • results[0].plot()：YOLO 內建一鍵繪製 bbox + ID + class + conf 的影像。
  • cv2.imshow：桌面除錯用，部署到無人機時通常關閉以節省資源。
  • cv2.VideoWriter：若需保存標註影片，可在每幀寫入輸出檔。
