無人機俯視拍攝

使用具備 GPS 定位與高度鎖定功能的無人機，沿著預先規劃的航線完成豬舍空拍。為確保後續演算法效果，建議遵守下列原則：

  • 飛行高度：5~10 公尺，視豬舍空間調整，避免擾動豬群但仍保留足夠解析度。
  • 拍攝角度：攝影機保持 90° 垂直向下，避免透視變形影響邊界框準確度。
  • 採樣率：4K 解析度搭配 30fps 錄影，事後抽幀作為訓練樣本。
  • 光照條件：避免正午過曝或陰影過深的時段，理想為晨間或多雲。

上傳至標註平台

常見的線上標註平台都支援 YOLO 格式匯出，可依團隊規模與預算選擇：

  • CVAT：開源、可自架，適合企業內部敏感資料，支援多人協作與審核流程。
  • Roboflow：SaaS 服務，提供資料增強、版本控制與 API，啟動門檻最低。
  • Label Studio：另一個開源選項，支援多種任務類型（不限影像）。

上傳後建立專案，將類別命名為 pig（編號 0）。

人工繪製邊界框

標註是整個 pipeline 中最費時但決定模型上限的步驟。建議：

  • 框緊邊緣：盡量貼合豬隻輪廓，避免大量背景被納入框內。
  • 遮擋處理：對於只露出部分身體的豬，仍需標註可見區域；若小於 20% 可忽略。
  • 標註一致性：多人協作時建立共同準則（SOP），定期交叉檢視。
  • 樣本數量：單類別建議至少 1,500~3,000 框作為起跑線。

匯出 YOLO 格式

YOLO 標籤檔格式為：[類別編號] [中心點X] [中心點Y] [寬度W] [高度H]，所有座標皆歸一化至 0~1：

  • 類別編號：整數，從 0 開始；本專案僅 0 = pig。
  • 歸一化座標：實際像素 ÷ 影像寬高，因此模型對解析度不敏感。
  • 檔案配對：每張 pic_001.jpg 必須有同名 pic_001.txt，缺檔將被忽略。
  • 空白標籤：若該張無目標物，仍可保留空 .txt 檔，YOLO 會視為「負樣本」用於降低誤判。

分割 train / val

將完成標註的影像依比例切分為訓練集與驗證集：

  • 典型比例：train : val = 80 : 20，資料量大時可拉到 90 : 10。
  • 隨機切分：避免時間或場景集中於同一邊，可使用 train_test_split 或 Roboflow 自動切分。
  • 不可洩漏：同一支影片抽幀後，應將整支影片分到同一邊，避免訓練/驗證重疊造成過度樂觀的指標。
  • 資料夾結構：嚴格遵守 train/images、train/labels、val/images、val/labels 的四層結構。

完成資料集

檢查清單：

  • 影像與標籤對齊：使用簡易腳本掃描每張圖片是否都有對應 .txt。
  • 視覺化抽查：隨機抽取 30~50 張將框畫回原圖，確認沒有座標錯位。
  • 類別分布：統計每張圖的目標數量分布，必要時補拍稀有場景。
  • dataset.yaml：準備設定檔，指向 train/val 路徑與類別表，將在階段 2 使用。

完成後，即可進入下一階段啟動模型訓練。
