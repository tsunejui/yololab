🐍 警報門檻判斷👤 後端工程師
def classify_severity(alerts):
    high_risk_keywords = [
        "逃脫", "中暑", "急性", "死亡"
    ]

    if alerts["ammonia_ppm"] > 30:
        return "critical"
    if alerts["count_drop_pct"] > 20:
        return "high"
    if any(k in alerts["summary"]
           for k in high_risk_keywords):
        return "high"
    return "normal"

📋 詳細資訊

取得 Agent 結論

Crew 完成後可取得多種形式的輸出：

  • result.raw：最後一個 Task 的純文字輸出（通常是 Markdown）。
  • tasks_output：每個 Task 的詳細結果，可分別取用。
  • json_dict：若 Task 設定 output_json，可直接取得結構化資料。
  • tokens / 成本：Crew 也會回傳本次執行的 token 用量，便於成本追蹤。

產生 Markdown 報告

為什麼選用 Markdown 作為中介格式？

  • LLM 友善：大模型對 Markdown 結構掌握度最高，輸出穩定。
  • 易於轉換：可一鍵轉為 HTML、PDF、DOCX，覆蓋所有交付場景。
  • 人類可讀：純文字即可閱讀，方便人工審核 AI 結論。
  • 版本控管：可入 Git 比對歷史報告變化，建立稽核軌跡。

轉換為 PDF

兩條主流路線：

  • WeasyPrint（Markdown → HTML → PDF）：用 CSS 控制版面，最容易做漂亮設計。
  • ReportLab（直接組裝）：程式化精細控制，適合需要動態圖表的報告。
  • 樣板套用：準備好公司 logo、頁首頁尾的 HTML/CSS 樣板，每次報告自動套用。
  • 圖表嵌入：用 matplotlib / plotly 產生圖表 PNG，插入 Markdown 一併渲染。

異常嚴重度判斷

決定推播強度與通知對象的關鍵邏輯：

  • normal：僅產生日報存檔，不主動推播。
  • high：推 Line 訊息 + 附 PDF 連結。
  • critical：Line + 電話 + 短訊三重通知，立刻找到負責人。
  • 關鍵字 + 數值門檻：同時用結構化欄位（氨氣 ppm）與 LLM 摘要關鍵字判斷，兩道防線。
  • 避免狼來了：對同一異常設定冷卻期，避免短時間重複轟炸。

推送 Line 通知

Line Bot 推送實務：

  • Push API：主動向使用者發訊息，需事先取得 user_id。
  • Flex Message：可發送卡片式訊息，內嵌按鈕跳轉 PDF 詳情。
  • 群組推播：農場主人 + 獸醫 + 員工建立群組，重要事件全員可見。
  • 多通道：除了 Line，也可同時發 Email、Slack、Telegram 增加觸達。
  • 送達回報：使用 webhook 確認訊息是否被讀取，未讀則升級通知。

報告送達農場主人

整套系統的最終交付：

  • PDF 報告：已上傳雲端儲存，可長期保留作為稽核紀錄。
  • Line 通知：農場主人手機即時收到摘要與行動建議。
  • 儀表板更新：後端同步更新 Web 儀表板，可隨時瀏覽歷史。
  • 形成資料閉環：農場主人處理結果可回饋給系統，作為下一輪 Agent 學習素材。
  • 系統運作完成 🎉：判別式 + 生成式 AI 整套智慧畜牧 pipeline 上線運行。
