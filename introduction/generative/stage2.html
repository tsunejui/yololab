🐍 LangChain 文件切割
from langchain.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

loader = PyPDFLoader("pig_disease_handbook.pdf")
docs = loader.load()

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50
)
chunks = splitter.split_documents(docs)
🐍 文字 → 向量
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small"
)

# 每個 chunk 被轉成 1536 維向量
vector = embeddings.embed_query(
    "豬隻氨氣濃度超標的警示症狀"
)
# → [0.0123, -0.0456, 0.0789, ...]
🐍 寫入 ChromaDB
from langchain_community.vectorstores import Chroma

vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./farm_kb"
)

vectorstore.persist()
# 知識庫已建立，可長期重複使用
🐍 相似度查詢
query = "3 號圈氨氣 25ppm 且豬隻數量異常下降可能原因"

results = vectorstore.similarity_search(
    query,
    k=3  # 取前 3 個最相關段落
)

for doc in results:
    print(doc.page_content)
    # → 「氨氣 > 20ppm 將刺激呼吸道...」
    # → 「夏季高溫常見群聚疊臥...」

📋 詳細資訊

蒐集知識文件

知識庫品質決定 RAG 的回答品質，建議來源：

  • 政府規範：農業部畜牧處的飼養標準、氨氣濃度上限等法規文件。
  • 學術文獻：豬隻疾病學教科書、最新的獸醫期刊論文。
  • 農場 SOP：自家累積的處置手冊、案例紀錄、巡檢清單。
  • 專家筆記：資深獸醫的口述經驗，整理為結構化文字。
  • 格式相容：支援 PDF、Word、Markdown、HTML，使用 LangChain 對應 Loader。

文件切割 Chunking

LLM 一次只能處理有限 token，文件切得越好，檢索越準：

  • chunk_size：每塊 300~800 字最常用；段落式知識可大一點。
  • chunk_overlap：相鄰塊重疊 10~15%，避免關鍵句被切斷。
  • RecursiveCharacterTextSplitter：依段落→句→字逐層分割，保留語意。
  • Markdown / HTML splitter：對結構化文件有專屬切割器，更聰明。

Embedding 向量化

把文字轉為高維向量，讓「語意相近」可被距離計算捕捉：

  • text-embedding-3-small：OpenAI 主力模型，1536 維，CP 值最高。
  • BGE / GTE：開源優質模型，可在本地運行降低成本。
  • 多語言：需處理中文時選用支援中文的模型（如 BGE-M3）。
  • 批次處理：多筆一起送 API 比逐筆呼叫便宜且快。

寫入向量資料庫

選型依規模而定：

  • ChromaDB：百萬筆以下、單機部署，最低門檻。
  • pgvector：已有 PostgreSQL 就直接擴充，省一套基礎設施。
  • Pinecone / Weaviate：千萬筆以上或需高可用，選託管雲服務。
  • metadata 過濾：同時存入分類標籤（疾病、規範、SOP），查詢時可加條件。

相似度檢索

查詢階段的關鍵技巧：

  • top-k：通常取 3~5 塊，太多會稀釋訊號、增加 token 消耗。
  • MMR (Max Marginal Relevance)：避免取到過度相似的重複內容。
  • Hybrid search：結合 BM25 關鍵字 + 向量檢索，補強純語意的盲點。
  • Re-ranking：用更小的 cross-encoder 模型對 top-20 重新排序，提升精度。

注入 Prompt Context

最終把檢索結果拼進 Agent prompt：

  • 清晰分隔：用 【背景知識】、【今日數據】等標籤區隔。
  • 引用來源：每塊加上 metadata（如「來源：豬隻疾病百科 p.45」），便於人工查核。
  • 避免幻覺：明確指示 LLM「僅根據以上知識回答，不確定請說明」。
  • 進入下一階段：已準備好高品質 context，交給 Agent 進行推理。
