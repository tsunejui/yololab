🐍 載入 LLM
import os
from crewai import Agent, Task, Crew, Process
from langchain_openai import ChatOpenAI

# 設定大語言模型大腦
os.environ["OPENAI_API_KEY"] = "your_api_key_here"

llm = ChatOpenAI(
    model="gpt-4o",
    temperature=0.2  # 低溫確保推理穩定
)
🐍 兩位 AI Agent
data_analyst = Agent(
    role='數據分析特工',
    goal='分析豬圈歷史數量與環境感測器數據，找出異常趨勢',
    backstory='你是一位精通統計學的 AI，'
              '專門從冰冷數據中抓出不尋常的警訊。',
    verbose=True,
    llm=llm
)

livestock_expert = Agent(
    role='資深畜牧專家顧問',
    goal='根據分析結果，提供畜牧建議與疾病預防方案',
    backstory='你擁有 30 年的養豬與獸醫經驗，'
              '深知環境氨氣、溫度與豬隻行為的因果關係。',
    verbose=True,
    llm=llm
)
🐍 任務鏈設計
task1 = Task(
    description=f'分析以下農場數據：{farm_data_json}。'
                '特別注意 16:00 數量從 50 隻降到 42 隻，'
                '以及氨氣濃度偏高的現象。',
    expected_output='一份條列式的異常與趨勢分析報告。',
    agent=data_analyst
)

task2 = Task(
    description='根據數據分析師的報告，'
                '撰寫專業營運建議書，包含：'
                '1. 數量突降的潛在原因分析；'
                '2. 環境改善指引。',
    expected_output='結構完整的 Markdown 農場日報。',
    agent=livestock_expert,
    output_file='pig_farm_daily_report.md'
)
🐍 組裝 Crew
pig_farm_crew = Crew(
    agents=[data_analyst, livestock_expert],
    tasks=[task1, task2],
    process=Process.sequential
    # 循序：分析師完成後交棒給專家
)

result = pig_farm_crew.kickoff()
print("## 報告生成完畢！")

設定 LLM 大腦

大模型選型影響成本與品質：

  • GPT-4o：推理頂尖、多模態能力強，適合主要決策節點。
  • Claude 3.5 Sonnet：長 context、結構化輸出穩定，適合冗長分析。
  • GPT-4o-mini / Haiku：簡單任務用便宜小模型，可大幅降低成本。
  • temperature：分析類任務設 0.1~0.3，避免創意發散；創意寫作可拉到 0.7+。
  • 本地 LLM：資料敏感時改用 Ollama + Llama 3 系列，完全離線。

定義 Agent 角色

好的角色設計是 Agent 表現的關鍵：

  • role：一句話標籤，例如「資深畜牧顧問」。
  • goal：明確的單一目標，避免模糊。
  • backstory：背景故事，影響語氣與專業視角。
  • 分工原則：讓每個 Agent 專精一件事（分析、撰寫、審核），互相補位。
  • tools：可指派外部工具（如 SQL 查詢、Web 搜尋）讓 Agent 自主呼叫。

設計 Task 任務

Task 是 Agent 的具體工作單元：

  • description：清楚描述輸入、目的與重點關注的指標。
  • expected_output：明示期望格式（條列、表格、Markdown），降低 LLM 隨意發揮。
  • output_file：自動將最終結果存檔，方便下游讀取。
  • context：可傳入前一個 Task 的輸出作為輸入，建構任務鏈。

組裝 Crew 工作流

Crew 串起多個 Agent 與 Task：

  • Process.sequential：循序執行，前者輸出自動接到後者輸入。
  • Process.hierarchical：有 manager Agent 動態分派任務，更複雜也更靈活。
  • memory=True：啟用記憶讓 Agent 在多輪互動間保留 context。
  • verbose：開發階段必開，可看到完整推理過程便於除錯。

啟動推理與反思

Agent 在 kickoff() 後內部執行多輪 ReAct 迴圈：

  • Thought：「需要先分析數量變化幅度」
  • Action：呼叫工具（如查 RAG 知識庫）
  • Observation：檢視工具回應內容
  • Critic / Reflexion：自我審查答案是否合理，發現問題再迭代
  • 監控成本：每輪都會呼叫 LLM，建議設定 max_iter 上限。

產出結構化結論

本階段交付：

  • 分析報告：由分析師 Agent 產出的異常摘要。
  • 建議報告：由專家 Agent 整合知識與分析後撰寫的決策建議。
  • 可追溯：verbose log 中保留完整推理鏈，可審核 AI 判斷邏輯。
  • 進入下一階段：把 Markdown 報告交給 Skills 模組轉成 PDF 並推播。
